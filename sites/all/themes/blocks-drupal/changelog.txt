===Blocks Changelog===

2.0.2 (07/08/13)
  *Fixed link color bug in color scheme style sheets.
  *Removed obsolete Portfolio page template and article/blog node tempaltes.
  *Organized template files into sub-folders.
  *Added options for menu descriptions and post "share icons".

2.0.1 (07/04/13)
  *Fixed major bug related to the Branding section.
  *Added several missing translation wrappers around text strings.
  *General style and bug fixes.

2.0 (07/01/13)
  *New "Tiles" content type and View block region.
  *Mega Slider Drupal module included ($20 value).
  *Removed page--front.tpl and all front page theme settings in favor of fully utilizing block regions and visibility settings.
  *Dynamically generated portfolio filters powered by taxonomy terms.
  *Added retina.js library.
  *Added Font Awesome icon font library.
  *Removed social icon settings in favor of using a block region.
  *Removed typography options in favor of module use.
  *Updated Twitter widget to work with API v1.1.
  *New Header block regions.
  *Menu now a block region instead of called programatically.
  *Moved code from /includes/header.php and /includes/footer.php into page template files.
  *Menu icons now controlled by menu descriptions, old JS will be left but commented out for legacy support.
  *Loading animation for front page.
  *Other minor style tweaks/fixes.

1.1 (05/20/13)
  *Updated Drupal Core to 7.22.
  *Changed branding link to "url('<front>')".
  *Removed theme jQuery library in favor of utilizing jQuery Update module.
  *Bricks sub-theme no longer requires parent CSS and JS files in sub-theme folder.
  *Removed theme SEO settings as modules offer superior functionality.
  *Removed unecessary Layout and Footer theme settings in favor of using block visibility settings
  *Updated documentation to reflect new changes.
  *Minor style fixes.

  *Updated Drupal Core to 7.21.

  *Fixed theme setting definition causing a bug in WAMP installs.
